# Module 03 - First Executable System Spec (ESS)

## Goal

Convert one valid issue into an implementation-ready execution spec.

## Time Box

- Target: 75 minutes

## Why this matters

A strong ESS aligns design, build, verification, and QA with less ambiguity.

## Required tasks

1. Create one ESS from template.
2. Define scope, non-goals, ACs, verification commands, rollback.
3. Run a 10-minute peer review.

## Stretch tasks

- Add a risk matrix with mitigation mapping.

## Build exercise

1. Copy `templates/ess-feature.md` to `docs/ess-issue-001.md`.
2. Fill all sections completely.
3. Make acceptance criteria binary and testable.
4. Add realistic rollback path.
5. Run peer review and capture comments.

## Micro checks

- Minute 25: scope and non-goals complete.
- Minute 45: ACs are binary and measurable.

## You should see

- ESS that another engineer can implement without a live clarification call.

## If this fails, do this

- remove vague language (fast, better, easier).
- replace with measurable outcomes and commands.

## Definition of done

- ESS passes peer review with no critical ambiguity.
