# 24 - Workshop Consistency Checklist

Use this file to keep the workshop practical and learner-friendly as you continue running it.

## Before You Start

Complete the modules in order through docs/23-curriculum-map-and-file-intent.md.

## How to use this file

Use this checklist before starting a module and after finishing it.

## Pre-run checklist

For the module you are about to run, confirm all are present:

1. Short context section
2. Immediate build task
3. Observable checkpoint
4. Clear next-step handoff

If one item is missing, patch the module first.

## In-run quality checks

During execution, confirm:

1. You are producing real artifacts, not only notes.
2. Decisions are recorded in issue history.
3. State and next gate are updated consistently.
4. Failed checkpoints are fixed before moving on.

## Post-run review

At session end, capture:

1. One artifact you created or improved
2. One bottleneck you hit
3. One change to improve the next run

Record these in your issue comments or working notes.

## File-type quick guide

Use this shortcut when choosing documents:

- Learning files: understand why
- Execution files: run steps now
- Artifact files: compare and validate output quality

## Success criteria for workshop quality

The workshop is functioning well when:

1. Learners can make progress without guessing hidden rules.
2. Every module creates a visible output.
3. Gate decisions are evidence-backed.
4. The next action is always clear.
