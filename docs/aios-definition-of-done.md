# AIOS Definition of Done

Your AIOS is considered operational when all are true:

1. Three consecutive work items completed end-to-end.
2. No skipped gates on any of those items.
3. Every item has full closure evidence links.
4. Branch protection enforces required checks.
5. Retry and escalation rules are used when failures occur.

## Performance Targets

- Median issue cycle time is stable or improving across last 3 items.
- QA rework rate is below 20 percent.

## Audit Targets

- Decision Log exists on every issue.
- Risk level is assigned and justified on every issue.
