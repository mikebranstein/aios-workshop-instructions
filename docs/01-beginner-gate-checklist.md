# Beginner Gate Checklist

Complete this checklist before starting 14-zero-to-hero-live-workshop.

## Purpose

This gate confirms you can run the workshop without hidden setup blockers.

## Tooling gate

- [ ] Git installed and `git --version` works
- [ ] VS Code installed and opens workspace
- [ ] .NET SDK installed and `dotnet --info` works
- [ ] GitHub Copilot and Copilot Chat installed in VS Code
- [ ] Copilot Chat returns JSON when requested

## Access gate

- [ ] Can fetch from remote repository
- [ ] Can create branch and push
- [ ] Can create GitHub issue
- [ ] Can create or edit GitHub Project board
- [ ] Can view branch protection rules or know who can apply them

## Process gate

- [ ] Understand State field as canonical source
- [ ] Understand one retry equals one full gate attempt
- [ ] Know escalation path for blocked issues
- [ ] Know closure requires ESS, PR, verification, QA links

## Dry-run gate

Run this quick dry run:

1. Create test issue.
2. Run intake prompt in Copilot Chat.
3. Capture JSON output.
4. Update board fields.
5. Add Decision Log comment.

Pass condition:

- All five actions complete in under 20 minutes without external help.

## Exit criteria

All checkboxes above are complete.

## Next step

Proceed to 14-zero-to-hero-live-workshop Day 1.
