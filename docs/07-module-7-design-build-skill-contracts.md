# Module 07 - Design and Build Skill Contracts

## Goal

Create reliable handoff between design and implementation.

## Time Box

- Target: 80 minutes

## Why this matters

Role specialization improves consistency when contracts are explicit.

## Required tasks

1. Define design skill contract.
2. Define build skill contract.
3. Run one issue through both and log traceability.

## Stretch tasks

- add explicit verification handoff fields.

## Build exercise

Create or update:
- `templates/skills/design-agent.md`
- `templates/skills/build-agent.md`

For each contract include:
- mission
- required inputs
- output schema
- guardrails
- escalation rule

Run both contracts on one issue and log references to ESS and acceptance criteria.

## Micro checks

- Minute 30: design contract complete.
- Minute 55: build contract complete.

## You should see

- design output references ESS; build output references acceptance criteria.

## If this fails, do this

- add missing references and rerun on the same issue.

## Definition of done

- handoff traceability is visible in issue timeline and artifacts.
