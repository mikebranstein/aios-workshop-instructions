# Module 09 - GitHub State as Source of Truth

## Goal

Keep project state, issue decisions, and evidence aligned.

## Time Box

- Target: 70 minutes

## Why this matters

Reliable state is required for automation, reporting, and recovery.

## Required tasks

1. Verify canonical project fields.
2. Perform state transitions on one active issue.
3. Reconcile state with timeline and PR evidence.

## Stretch tasks

- add a dashboard view filtered by `Next Gate`.

## Build exercise

Verify or create fields (reuse existing first):
- State
- Risk
- Next Gate
- Owner Agent

Run one issue through at least three transitions and validate alignment:
- project item state
- issue decision comments
- PR/check evidence

## Micro checks

- Minute 20: fields verified.
- Minute 45: first transition sequence complete.

## You should see

- consistent state story across project board, issue timeline, and PR evidence.

## If this fails, do this

- correct the mismatch and post one reconciliation note.

## Definition of done

- one issue has synchronized state and evidence trail.
