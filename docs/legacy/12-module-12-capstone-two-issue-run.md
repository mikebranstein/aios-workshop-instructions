# Module 12 - Capstone: Two Back-to-Back Issue Runs

## Goal

Demonstrate repeatable quality and control across two full issue lifecycles.

## Time Box

- Target: 90 minutes per issue run (two runs)

## Why this matters

A single successful run can be luck. Two consistent runs show operational maturity.

## Required tasks

1. Run two issues end-to-end using the full workflow.
2. Capture metrics for each run.
3. Complete one postmortem and one improvement item.

## Stretch tasks

- propose one automation improvement based on run comparison.

## Build exercise

For each issue:
1. intake decision
2. ESS
3. design and build execution
4. verification evidence
5. QA report
6. PR and closure evidence

After both runs:
- compare cycle time
- compare retries
- identify primary blocker patterns
- create one backlog improvement item

## Micro checks

- Run 1 midpoint: verification evidence posted.
- Run 2 midpoint: QA report posted.

## You should see

- two completed issues with full evidence and no skipped gates.

## If this fails, do this

- stop at first failing gate, perform root-cause fix, and rerun affected segment.

## Definition of done

- two successful runs with complete evidence and policy compliance.
- postmortem report and improvement backlog item created.
