# Module 04 - Objective Gate Contracts

## Goal

Define quality gates so decisions are evidence-based and repeatable.

## Time Box

- Target: 70 minutes

## Why this matters

Objective gates prevent subjective pass/fail judgments.

## Required tasks

1. Define Design, Verification, QA, and Merge gate criteria.
2. Add required evidence for each gate.
3. Run a maker-checker review.

## Stretch tasks

- Add edge-case fail signals per gate.

## Build exercise

Update `docs/gates.md`:
- each gate has required inputs
- each gate has objective pass criteria
- each gate has fail signals
- each gate has required evidence links

Run a two-person review on one sample issue and compare decisions.

## Micro checks

- Minute 20: Design and Verification gates complete.
- Minute 40: QA and Merge gates complete.

## You should see

- two reviewers reach the same decision from the same evidence.

## If this fails, do this

- remove subjective wording.
- tighten criteria into measurable checks.

## Definition of done

- inter-rater agreement achieved on at least one sample issue.
